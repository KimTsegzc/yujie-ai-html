# 长期记忆 · 完整演示版项目

## 项目规范（必须遵守）

### 1. 字体体系强制规则
- **所有文字必须使用v17字体变量**，禁止硬编码 `font-size: XXpx`
- **最低可读标准**：投影场景下，20米外的观众必须能看清
- **正文最小字号**：`--current-base`(16px) 或以上，绝对不用 `--text-sm`(12.8px) 作为正文
- **每次修改完文件后，必须自检字体大小**
- 字体变量层级（从小到大）：`--text-sm`(12.8,仅辅助标注) → `--current-base`(16,标准正文) → `--text-h4`(25) → `--text-lead`(20) → `--text-body`(16) → `--text-h3`(31.3) → `--text-h2`(39.1) → `--text-h1`(48.8)

### 2. 版本管理规范
- 每次修改HTML文件前，确认当前版本号
- 重大功能改动需要记录版本变更日志到当日memory文件
- 修改前理解现有逻辑再动手，避免引入回归bug

### 3. 观众友好原则
- 投影演示场景是主要使用场景
- 所有文字、数据、标签都必须在20米距离可读
- 交互元素（按钮/卡片）要有足够的视觉对比度和点击区域
- 对话/话术内容要丰富、人性化、有业务价值，不能太单薄

### 4. 动画时序规范
- 工作区动画必须在用户发送对话指令之后才触发
- 正确流程：进入步骤→等待→用户发消息→对话播放→工作区内容生成
- 禁止一进入页面就自动展示工作区内容

## 已知项目结构
- 主入口：完整演示版v8.html（iframe壳）
- theme-config.js：6套颜色主题中央管理
- 子页面通过postMessage同步主题切换
- v17等比缩放字体系统：18px基准，Perfect Fourth 1.333比例

## 前台vs后台交互差异
- 前台（03/04）：对话驱动(60%)+轻工作区(40%)，iPad场景，卡片式输出
- 后台（05）：上传驱动+重工作区(70%)+轻对话(30%)，桌面大屏，左文字+右图表双栏
- 后台技术栈：SheetJS(CDN) + ECharts(CDN)，不依赖本地JS

## 调试经验教训
- **DOM操作必须做空值保护**：任何`getElementById()`后接`.xxx`调用时，先判断是否为null
- **副作用函数不要放在核心逻辑之前**：动画函数崩溃会连带杀死核心交互代码
- **"逻辑正确但功能不出现"的排查方法**：最小化测试页验证纯逻辑 → 日志覆盖层定位具体中断行
